Fix two critical bugs in the authentication and routing system:

═══════════════════════════════════════
BUG 1 — LOGIN TYPE VALIDATION
═══════════════════════════════════════
Currently any email can login under any tab (Personal/Dokter Institusi/Admin Institusi).
This must be enforced strictly based on doctor_accounts data.

After successful supabase.auth.signInWithPassword:
1. Fetch profile from doctor_accounts where user_id = auth user id
2. Apply strict validation:

   IF loginType === 'personal':
   - profile.enterprise_id must be NULL
   - If profile.enterprise_id is filled → reject with error:
     "Akun ini terdaftar sebagai akun institusi. Pilih tab Institusi."

   IF loginType === 'institusi' + subType === 'dokter':
   - profile.enterprise_id must NOT be null
   - profile.role must be 'doctor'
   - If enterprise_id null → reject: "Akun ini bukan akun institusi. Pilih tab Personal."
   - If role is 'admin' → reject: "Akun ini adalah Admin Institusi. Pilih Admin Institusi."

   IF loginType === 'institusi' + subType === 'admin':
   - profile.enterprise_id must NOT be null
   - profile.role must be 'admin'
   - If enterprise_id null → reject: "Akun ini bukan akun institusi. Pilih tab Personal."
   - If role is 'doctor' → reject: "Akun ini adalah Dokter Institusi. Pilih Dokter Institusi."

3. Only proceed with onLogin after ALL validations pass

═══════════════════════════════════════
BUG 2 — ADMIN INSTITUSI ROUTING
═══════════════════════════════════════
Admin Institusi currently shows the same view as doctor (Dashboard).
It must route to AdminDashboard instead.

In App.tsx, update handleLogin:
- If role === 'admin' AND plan === 'enterprise':
  → setCurrentView('admin-dashboard')
- If role === 'doctor':
  → setCurrentView('dashboard')

In AdminDashboard.tsx, this is the institutional admin view showing:
- List of all doctors in their institution (filtered by enterprise_id)
- Manage seats/licenses for their institution
- View usage statistics for their institution
- Add/remove doctors within their institution
- They CANNOT see other institutions' data
- They CANNOT access Aexon-level admin features

Make sure AdminDashboard receives the enterprise_id from userProfile
so it can filter doctors by institution:
- Add enterprise_id to UserProfile type in types.ts
- Pass enterprise_id from doctor_accounts profile to userProfile on login
- In AdminDashboard, filter doctors list by enterprise_id
