In Settings.tsx, fix the Langganan tab pricing modal.

Currently it shows ALL 4 plans (beta monthly, beta annual, 
normal monthly, normal annual) at once. This is wrong.

Fix: fetch beta_mode from Supabase app_settings, then only show
2 plans (monthly + annual) based on whether beta is active or not.

1. ADD beta mode fetch in the useEffect that fetches plans:

useEffect(() => {
  async function fetchPlansAndSettings() {
    if (!supabase) { setPlansLoading(false); return; }
    try {
      // Fetch beta mode
      const { data: betaData } = await supabase
        .from('app_settings')
        .select('value')
        .eq('key', 'beta_mode')
        .maybeSingle();
      
      const isBetaMode = betaData?.value === 'true';
      
      // Fetch ALL plans
      const { data: allPlans } = await supabase
        .from('product_plans')
        .select('*, products(name)')
        .order('price', { ascending: true });
      
      if (allPlans) {
        // Filter: only show beta plans if beta_mode=true,
        // only show normal plans if beta_mode=false
        // Beta plans = have original_price (discounted)
        // Normal plans = original_price is null
        const filtered = allPlans.filter(p => 
          isBetaMode 
            ? p.original_price !== null  // beta = has strikethrough price
            : p.original_price === null  // normal = no strikethrough
        );
        setPlans(filtered);
      }
    } catch (err) {
      console.error('Failed to fetch plans:', err);
    } finally {
      setPlansLoading(false);
    }
  }
  fetchPlansAndSettings();
}, []);

- When beta_mode = false → shows Rp799k/bln and Rp649k/bln
- Always shows exactly 2 plans (monthly + annual)
