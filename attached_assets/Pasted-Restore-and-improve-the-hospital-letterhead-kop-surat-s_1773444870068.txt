═══════════════════════════════════════
KOP SURAT — PERSONAL ACCOUNT
═══════════════════════════════════════
In Settings.tsx, restore the hospital settings section for personal accounts.

RULES:
- Maximum 3 kop surat (dokter personal maksimal 3 tempat praktik)
- Each kop surat has these fields:
  • Nama RS / Klinik (subject to 14-day cooldown)
  • Logo RS (subject to 14-day cooldown)  
  • Alamat (no cooldown)
  • No. Telepon (no cooldown)
  • No. Fax (no cooldown)
  • Email RS (no cooldown)
  • Website RS (no cooldown)

COOLDOWN LOGIC:
- When doctor saves changes to "Nama RS" or "Logo RS":
  Store in localStorage: 'aexon_kop_cooldown_[userId]_[kopIndex]' = 
  JSON.stringify({ last_changed: new Date().toISOString() })
- On load: check if (now - last_changed) < 14 days
  → If YES: disable Nama RS and Logo fields, show:
    "Nama dan logo dapat diubah kembali pada [tanggal]"
    with a small CalendarClock icon (lucide-react)
  → If NO: fields are editable normally
- Minor fields (alamat, telp, fax, email, website): always editable, no cooldown

UI FOR PERSONAL KOP SURAT (in Settings.tsx PROFIL tab or separate "Kop Surat" tab):
- Section title: "Kop Surat Praktik"
- Show up to 3 kop cards, each collapsible
- Each card header: Nama RS (or "Kop Surat 1/2/3" if empty), edit icon
- "Tambah Kop Surat" button (only shows if less than 3 kop exist)
- "Hapus" button on each card (with confirm modal)
- Each card form fields as listed above
- Cooldown indicator per card if applicable
- "Simpan" button per card

STORAGE (localStorage for Fase 1):
- Key: 'aexon_hospital_settings_[userId]' = JSON array of up to 3 kop objects
- Each object: { id, nama, logo_url, alamat, telp, fax, email, website, last_name_changed, last_logo_changed }
